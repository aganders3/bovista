bovista::scene

Struct Scene 

Source 
pub struct Scene { /* private fields */ }
Expand description

A scene contains a collection of visual objects to be rendered.

The scene manages the lifetime of all visual objects and coordinates their preparation (updating GPU buffers) and rendering phases. Visuals are stored in insertion order and rendered in that same order.

§Platform Differences

On native platforms, visuals are stored in Arc<Mutex<>> for thread-safety. On WASM, where threads aren’t available, Rc<RefCell<>> is used instead.

§Example

use bovista::{Scene, Points};
use std::sync::{Arc, Mutex};

let mut scene = Scene::new();

// Add visuals to the scene
let points = Points::test_cube(&device, &format, &layout, 10);
let index = scene.add(Arc::new(Mutex::new(points)));

// Prepare and render
scene.prepare(&device, &queue);
scene.render(&mut render_pass);

// Remove a visual
scene.remove(index);

pub fn render(&self, render_pass: &mut RenderPass<'_>)

Render all visible visuals in the scene. The renderer is expected to have already bound the camera bind group at slot 0; every visual reads it via @group(0) and none of them overrides slot 0 themselves.
