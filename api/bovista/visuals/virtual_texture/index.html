Constants§

MAX_ATLAS_COUNT
Maximum number of physical atlas textures a single visual can hold. Visuals always pre-bind this many atlas textures in their shader bind group; visuals that request fewer get 1×1×1 dummy textures in the unused slots. The page-table encoding reserves 2 bits for atlas_id, so bumping this above 4 requires widening that field (see page_table.rs).

Functions§

wanted_sorted
Snapshot of the wanted set as (lod_level, t, z, y, x, priority) tuples, sorted by priority (lower = more urgent; 0 = “user is looking at this t now”, positive = prefetch offset). Shared by every language binding so the sort + projection lives in one place.
