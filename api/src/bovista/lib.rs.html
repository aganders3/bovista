1//! # Bovista
2//!
3//! A high-performance 3D visualization library built on WebGPU for rendering large-scale
4//! volumetric imaging data with on-demand chunked loading.
5//!
6//! ## Architecture Overview
7//!
8//! Bovista is designed around a scene graph architecture with the following key components:
9//!
10//! ### Core Components
11//!
12//! - **[`Renderer`]**: Manages GPU resources, render pipelines, and coordinates frame rendering
13//! - **[`Scene`]**: Container for visual objects that implements visibility culling and rendering order
14//! - **[`Camera`]**: Perspective camera with orbit, zoom, and projection controls
15//! - **[`Visual`]**: Base trait for all renderable objects in the scene
16//!
17//! ### Visual Types
18//!
19//! Bovista provides several built-in visual types for different use cases:
20//!
21//! - **[`Points`]**: Render point clouds with per-point colors
22//! - **[`Lines`]**: Render line segments and wireframes
23//! - **[`Image`]**: Render 3D volumes with slice plane visualization and on-demand chunk loading
24//! - **[`Custom`]**: Base for creating custom visualization types
25//!
26//! ### Chunked Loading System
27//!
28//! [`Image`] supports chunked loading with multi-resolution LOD (Level of Detail) for handling
29//! datasets that are too large to fit in memory:
30//!
31//! 1. **Spatial Partitioning**: Volume is divided into tiles at multiple LOD levels
32//! 2. **Visibility Culling**: Only visible tiles are loaded based on frustum and LOD selection
33//! 3. **LRU Cache**: Least recently used tiles are evicted when memory limit is reached
34//! 4. **Async Loading**: Tiles are loaded asynchronously via callbacks (typically from Python)
35//!
36//! ## Usage Example
37//!
38//! ```rust,no_run
39//! use bovista::{Renderer, Scene, Camera, Points};
40//!
41//! // Initialize renderer with GPU device
42//! let renderer = Renderer::new(device, queue, surface_format).await;
43//! let mut scene = Scene::new();
44//! let camera = Camera::new(aspect_ratio);
45//!
46//! // Create and add visuals
47//! let points = Points::test_cube(&device, &format, &layout, 10);
48//! scene.add(Arc::new(Mutex::new(points)));
49//!
50//! // Render loop
51//! renderer.update_camera(&camera);
52//! scene.prepare(&device, &queue);
53//! renderer.render(&scene, &view, &depth_view, clear_color);
54//! ```
55//!
56//! ## Python Bindings
57//!
58//! When compiled with the `python` feature, Bovista provides Python bindings via PyO3:
59//!
60//! ```python
61//! import bovista as bv
62//! import numpy as np
63//!
64//! # Create viewer
65//! viewer = bv.PyViewer(800, 600)
66//! viewer.initialize()
67//!
68//! # Add point cloud
69//! positions = np.random.rand(1000, 1, 3).astype(np.float32)
70//! colors = np.random.rand(1000, 1, 3).astype(np.float32)
71//! points = bv.PyPoints.from_numpy(viewer, positions, colors)
72//! viewer.add_points(points)
73//!
74//! # Run interactive window
75//! viewer.run()
76//! ```
77//!
78//! ## Remote Zarr Example
79//!
80//! The most advanced usage is loading multiscale OME-Zarr data from remote S3:
81//!
82//! ```python
83//! import zarr
84//! import bovista as bv
85//!
86//! # Open remote zarr store
87//! store = zarr.open("https://s3.../dataset.zarr", mode="r")
88//! array = store["0"]  # Resolution level 0
89//!
90//! # Define tile loader callback
91//! def load_chunk(lod_level, tile_z, tile_y, tile_x):
92//!     tile_data = array[tile_z:tile_z+tz, tile_y:tile_y+ty, tile_x:tile_x+tx]
93//!     return np.array(tile_data, dtype=np.uint8)
94//!
95//! # Create image with chunked loading and multiple LOD levels
96//! image = bv.Image.from_chunked(
97//!     viewer,
98//!     lod_levels=[...],  # List of LevelMetadata for each LOD
99//!     loader=load_chunk,
100//!     max_chunks=500
101//! )
102//! ```
